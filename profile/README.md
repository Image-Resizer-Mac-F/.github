## 📋 Overview

<a href="#imageResizer">Image Resizer</a> is a batch image processing application for Mac that resizes, converts, and prepares large quantities of images efficiently — replacing the manual image-by-image processing that photo editing applications require for tasks that need the same operation applied to many images simultaneously.

The <a href="#imageResizer">batch processing engine</a> applies configured resize and conversion operations to any number of images in a single run — selecting hundreds or thousands of images, configuring the output dimensions and format, and processing the entire set without manual intervention. Tasks that would require hours of individual file processing in an image editor complete in minutes with batch processing, making image preparation workflows practical at scale. The <a href="#imageResizer">flexible dimension specification</a> supports multiple approaches to defining output size — exact pixel dimensions, percentage of original size, maximum width or height constraints, fit within dimensions, fill dimensions, and other sizing modes that cover the different requirements of web, print, and device image preparation. Each mode handles the size calculation differently, making the correct mode available for whatever the specific preparation task requires.

The <a href="#imageResizer">format conversion</a> changes image format during the batch operation — converting a folder of PNG screenshots to compressed JPEGs for web, converting TIFF print files to web-optimized WebP, or converting any source format to the target format that the destination workflow requires. Format conversion with quality control specifies the compression level for lossy formats — balancing file size against visual quality for the specific use case. The <a href="#imageResizer">output organization</a> saves processed images to configurable output locations — a specific output folder, subfolders mirroring the source structure, or alongside originals with name suffixes that distinguish the processed versions from the originals.</p>

<a href="#imageResizer">EXIF metadata handling</a> configures whether the processed images preserve the original camera metadata — preserving date, camera, GPS, and other EXIF data that downstream workflows need, or stripping metadata for privacy-sensitive publishing scenarios where location and device information shouldn't accompany the image. <a href="#imageResizer">Watermark overlay</a> adds text or image watermarks to processed images — positioning a logo or copyright notice on every processed image as part of the batch operation without manual placement in each individual file.

<a href="#imageResizer">Quality and compression control</a> sets the output quality for JPEG and other lossy formats — fine-tuning the file size to quality trade-off for the specific delivery requirement. <a href="#imageResizer">Drag-and-drop workflow</a> accepts image files and folders dropped directly onto the application for processing — loading images without navigating file dialogs for the quick-processing use cases that come up repeatedly in image-heavy workflows.</p>
